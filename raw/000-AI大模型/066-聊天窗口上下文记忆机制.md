### 一、**基础机制：上下文窗口的限制与应对**
1. **上下文窗口（Context Window）**  
   - 模型在生成每个新token时，仅能参考窗口范围内的历史信息（如GPT-4 Turbo支持128K token窗口）[citation:6][citation:8]。
   - **问题**：窗口外的信息会被“遗忘”，例如ChatGPT应用端仅保留最近4,096 token，超出部分无法被利用[citation:6][citation:7]。
   - **优化策略**：
     - **滑动窗口**：仅保留最近几轮对话（如最近5轮），丢弃早期内容[citation:5][citation:8]。
     - **Token限制裁剪**：动态截断历史对话，确保输入长度不超过模型上限[citation:3][citation:5]。

---

### 二、**长期记忆的核心技术方案**
#### 1. **检索增强生成（RAG + 向量数据库）**
   - **流程**：
     1. 将历史对话转化为向量嵌入（如OpenAI Embedding）。
     2. 存储至向量数据库（如Weaviate）。
     3. 根据当前输入检索相关记忆片段，拼接至提示词中[citation:3][citation:2]。
   - **案例**：  
     用户说“我住在北京” → 存入向量库 → 用户问“北京天气？” → 检索“北京”记忆并生成回答[citation:3]。

#### 2. **显式记忆管理框架（如Mem0、LangChain）**
   - **功能**：
     - 自动分类记忆（短期/长期/用户偏好）。
     - 动态检索与插入相关记忆到提示词。
   - **伪代码示例**：
     ```python
     memory = Memory()
     memory.add(history, user_id="user1")  # 存储对话
     relevant = memory.search("北京天气")   # 检索相关记忆
     prompt = format_prompt(relevant, current_input)
     response = llm.call(prompt)
     ```
     通过框架管理多轮对话的核心信息，避免冗余[citation:3][citation:5]。

#### 3. **摘要压缩（Summarization）**
   - 将长对话历史压缩为简短摘要，保留关键信息。
   - **示例**：  
     将10轮对话压缩为300字的摘要，作为后续对话的上下文[citation:1][citation:8]。
   - **优势**：节省70%以上Token，维持长期连贯性[citation:8]。

#### 4. **动态记忆模块（生物启发机制）**
   - **记忆网络（Memory Networks）**：  
     外部存储矩阵动态读写关键信息（如用户偏好）[citation:1][citation:4]。
   - **神经图灵机（NTM）**：  
     控制器按需访问外部记忆库，模仿计算机存储机制[citation:4]。
