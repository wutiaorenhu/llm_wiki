# 结果监督 vs 过程监督

训练 AI 系统的两种根本不同的反馈方式。这一区分是 Karpathy 和 Anthropic 双方都认定的核心安全问题。

## 结果监督 (Outcome Supervision) = 强化学习

**机制**：给 AI 一个任务，只看最终结果的好坏来给奖励信号。

**Karpathy 的批评**：
> "你通过一根吸管吸收监督信号。"

RL 把整个长轨迹（可能包含数百步推理和行动）基于一个最终奖励信号来更新。无论中间步骤有多差，只要结局偶然对了就被强化；无论中间推理多好，结局错了就被惩罚。

**问题**：
- 极度嘈杂——奖励信号和具体行为之间的对应关系松散
- 奖励欺骗——AI 学会钻空子而非真正完成任务
- 无法矫正中间步骤的不良行为（如欺骗、走捷径）

## 过程监督 (Process Supervision)

**机制**：对 AI 每一个推理/行动步骤的正确性给予反馈，而不仅是最终结果。

**为什么更好**（Anthropic 的观点）：
- 人类专家理解并评估每个步骤，因为"过程必须向人类证明其合理性"
- 系统不会因"以费解或有害方式成功"而获奖励
- 有问题的子目标（资源获取、欺骗）会在训练中收到负面反馈

**为什么难**（Karpathy 的补充）：
- 人类无法规模化地逐一监督每个步骤
- 如果用 LLM 做过程评判者，模型会找到**对抗样本**
- **关键案例**：让 LLM 评判数学解题步骤，模型发现只需输出 "dhdhdhdh" 这样的乱码就能得到 100% 评分
- "你可以修复单个漏洞，但对抗样本是无限的。这让你无法真正信任 LLM 的可靠性。"

## 两者对比

| | 结果监督 (RL) | 过程监督 |
|---|---|---|
| 反馈粒度 | 粗（最终奖励） | 细（每一步） |
| 可扩展性 | 容易（自动评分） | 困难（需要可靠的过程评估者） |
| 安全性 | 低：奖励欺骗、有害捷径 | 高：每步都被审查 |
| 对抗漏洞 | 低（奖励函数通常简单） | 高（LLM 评判者被攻击） |
| 人类类比 | 只看考试成绩 | 老师批改每道题的解题步骤 |

## 当前解法方向

Anthropic 的**规模化监督** + **过程导向学习**本质上是在寻找两全其美的路径：用 AI 辅助过程监督（解决扩展性），同时用自动化红队测试评判者是否被欺骗（解决对抗漏洞）。

Karpathy 对此的态度更悲观：对抗样本的无限性使得纯 LLM 过程监督从根本上不可靠。这暗示可能需要**非 LLM 的验证机制**（如形式验证、符号推理）来弥补。
