# 认知核心 (Cognitive Core)

Karpathy 提出的模型架构愿景：将 AI 系统中的**推理能力**与**记忆/知识**分离，只保留一个最精简的推理引擎。

## 核心理念

当前 LLM 的权重同时承载两件事：
- **算法性智能**（如何推理、如何思考）
- **事实性知识**（谁、什么、何时、何地）

Karpathy 认为这应该拆分：
- 认知核心只负责推理 —— 需要尽可能小
- 事实知识通过**工具检索**获取 —— 不在权重中存储

## 规模估计

- 推测约 **10 亿参数**可能够支撑有意义的认知
- "至少需要十亿个旋钮才能产生有趣的认知"
- 在当前蒸馏趋势下，可能还能再小一点，但不太可能远低于此

## 架构类比

| 组件 | 大脑类比 | 功能 |
|---|---|---|
| 认知核心（权重） | 皮层组织 | 算法性推理 |
| KV cache / 上下文 | 工作记忆 | 当前意识、正在处理的 token 流 |
| 外部检索 | 海马体 + 外部世界 | 查事实、读文档、调用工具 |

## 与记忆的关系

Karpathy 认为 LLM 的**记忆能力太强是个缺陷**：
- 能完整记住随机序列并非好事
- 应该"被迫去查找东西"，像人类依赖外部记忆一样
- 减少模型内部的记忆容量，强制依赖检索和工具

## 与当前研究的关联

- 与 Anthropic 的**机械可解释性**互补：如果能把认知核心理解透（逆向工程），就可以更自信地构建更大系统
- 与 **Wiki 方法论**高度一致：Wiki 是外部知识存储，LLM 是认知核心——不在权重中存储知识，而是读文件来获取
- 本质上是"小模型 + 大知识库"的范式，区别于"大模型记住一切"的当前主流

## 时间线

Karpathy 认为这大约需要 **20 年**来充分实现。近期（10 年内）主流仍是"用梯度下降训练的大型神经网络"，各方面逐步 +20% 的改进。
