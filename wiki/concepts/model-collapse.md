# 模型崩溃 (Model Collapse)

当模型在自己生成的数据上进行训练时，输出分布的多样性急剧坍缩到极窄范围的现象。

## 现象

在合成数据上迭代训练多代模型时：
- 每一代模型输出多样性下降
- 输出的"尾部"（罕见但重要的模式）首先消失
- 最终所有输出趋向同质化——分布坍缩到几个高频模式
- Karpathy 称之为"静默崩溃"（silently collapsed）——不容易被单个指标察觉

## Karpathy 的类比：人类也会坍缩

> "孩子还没有过拟合。成年人已经坍缩了。"

- 儿童思维灵活，未被固定模式束缚
- 随着年龄增长，人越来越依赖已有的思维模式
- 成年人在某种意义上已经"过拟合"到自己的经验上
- 创造性工作本质上是在对抗这种认知坍缩

## 与人类学习的联系

这是 Karpathy 解释"人类如何学习"的核心对比：

| | LLM | 人类 |
|---|---|---|
| 输入 | 文本 | 文本 |
| 处理 | 下一个 token 预测 | 内部反思、联想、重组、"合成数据生成" |
| 输出 | 下一个 token 概率分布 | 经过重构的理解和新想法 |
| 结果 | 记住训练数据的模式 | 生成自己的心智模型 |

> "书是一套提示，让我去做合成数据生成。"

人类在阅读时不只是在预测下一个词——我们停下来想、在脑中重新组织、用自己的话总结、与已有知识建立关联。这个**反思阶段**是 LLM 当前缺失的，也是人类学习不崩溃的关键。

## 为什么对 AI 安全重要

- 如果 AI 生成的文本在网上越来越多，训练数据将包含大量合成数据
- 模型崩溃可能被误解为"AI 变安全了"——实际上只失去了多样性和创造力
- 解锁反思式学习可能是防止模型崩溃的关键
- 与"认知核心"构想联系：如果模型通过检索而非记忆来获取知识，合成数据的污染影响会减少
