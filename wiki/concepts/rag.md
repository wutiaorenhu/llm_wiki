# 检索增强生成 (RAG)

RAG (Retrieval-Augmented Generation) 是将外部知识检索与 LLM 生成相结合的技术范式，解决大模型的知识截止、幻觉和溯源问题。

## 核心流程

1. **文档解析** — 结构化提取（表格、段落、图片）
2. **文本分块** — chunk 策略影响检索粒度
3. **向量嵌入** — 将文本映射到语义向量空间
4. **相似检索** — 查询嵌入与文档嵌入的最近邻匹配
5. **增强生成** — 检索结果注入 prompt 上下文

## 关键技术挑战

### 表格解析
- 传统分块易割裂表格结构，需保留行列关系
- 跨表格语义关联需显式建模
- 大表格向量嵌入导致维度爆炸

### Graph RAG
- 基于知识图谱的结构化检索
- 实体-关系-实体三元组建模
- 解决多跳推理问题

### RAG vs 微调
- RAG：外部知识注入，可解释、可更新、防幻觉
- 微调：知识内化到模型参数，效率高但有遗忘风险
- 两者互补，非替代关系

## 与传统向量搜索的关系

RAG 是上层应用范式，向量搜索是其核心检索组件。区别在于 RAG 还包括文档解析、分块策略、prompt 集成等完整 pipeline。
